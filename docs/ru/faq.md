# Частые вопросы

## Почему недоступно избранное учётной записи?

KToolBox v1 реализует 14 операций Pawchive, не требующих входа. Пять операций OpenAPI, защищённых `cookieAuth`, намеренно исключены, поэтому клиент API никогда не принимает и не отправляет сеанс учётной записи.

Отметка публикации — отдельная публичная операция, и она реализована. Вызывайте её осознанно: успешная отметка меняет состояние сервера, а уже отмеченная публикация может вернуть `PawchiveConflictError`.

## Что делать при ошибке вызова API?

CLI сообщает типизированные ошибки Pawchive, а не возвращает частично разобранный ответ. Основные классы: транспорт, HTTP, аутентификация, отсутствие ресурса, конфликт и проверка ответа.

- Проверьте URL или правильность `service`, ID автора и ID публикации.
- Увеличьте `KTOOLBOX_API__TIMEOUT` для медленного соединения.
- Настройте `KTOOLBOX_API__RETRY_TIMES` и `KTOOLBOX_API__RETRY_INTERVAL` для временных ошибок транспорта, `429` и `5xx`.
- Не добавляйте cookie учётной записи в настройки API: запросы API его не используют.

Перенаправления, обычные ответы `4xx`, конфликты и неверные данные ответа не повторяются.

## Почему при скачивании файлов возвращается 403?

Если файловому серверу для конкретного ресурса нужен сеанс, задайте ключ только для загрузчика:

```dotenv
KTOOLBOX_DOWNLOADER__SESSION_KEY=xxxxx
```

Cookie ограничен файловыми запросами и не отправляется в API Pawchive. Считайте `.env` и `prod.env` секретными локальными файлами и не фиксируйте их в репозитории.

## Как продолжить прерванную загрузку?

Снова выполните ту же команду. KToolBox пропускает завершённые целевые файлы. Если существует незавершённый файл с `downloader.temp_suffix` и сервер поддерживает диапазоны, загрузчик запрашивает оставшиеся байты и проверяет итоговый размер.

## Как отключить обложки или вложения?

```dotenv
# Только вложения.
KTOOLBOX_JOB__DOWNLOAD_FILE=False
KTOOLBOX_JOB__DOWNLOAD_ATTACHMENTS=True

# Только обложки.
#KTOOLBOX_JOB__DOWNLOAD_FILE=True
#KTOOLBOX_JOB__DOWNLOAD_ATTACHMENTS=False
```

`download_file` означает основной файл публикации, обычно обложку. Оба параметра по умолчанию равны `True`.

## Можно ли хранить вложения прямо в каталоге публикации?

```dotenv
KTOOLBOX_JOB__POST_STRUCTURE__ATTACHMENTS=./
```

## Как избежать слишком длинных имён файлов?

Используйте последовательные имена или ограничение точности формата:

```dotenv
KTOOLBOX_JOB__SEQUENTIAL_FILENAME=True
KTOOLBOX_JOB__POST_DIRNAME_FORMAT=[{published}]{id}_{title:.30}
KTOOLBOX_JOB__FILENAME_FORMAT={title:.30}_{}
```

## Как настроить прокси?

HTTPX читает стандартные переменные окружения прокси:

```bash
export HTTPS_PROXY=http://127.0.0.1:7897
export HTTP_PROXY=http://127.0.0.1:7897
export ALL_PROXY=socks5://127.0.0.1:7897
```

В PowerShell:

```powershell
$env:HTTP_PROXY="http://127.0.0.1:7897"
$env:HTTPS_PROXY="http://127.0.0.1:7897"
```

## Почему не открывается редактор конфигурации?

Установите необязательную зависимость терминального интерфейса:

```bash
pip install "ktoolbox[urwid]"
# или
pipx install "ktoolbox[urwid]" --force
```

## Как регулярно синхронизировать много авторов?

Добавьте каждого автора в список проекта, при желании назначьте короткий псевдоним, затем выполняйте `sync` без целей:

```bash
ktoolbox creator add fanbox:123 --alias studio-a
ktoolbox creator add patreon:456 --alias studio-b
ktoolbox sync
```

`creator disable` сохраняет запись, но исключает её из запусков без целей. Отключённый псевдоним по-прежнему можно синхронизировать явно. Подготовка авторов и передача файлов имеют отдельные ограничения параллельности, поэтому крупный автор не занимает все готовые загрузки.

## Как исключить разные темы для разных авторов?

Добавьте отдельные записи `[[blockers]]` в `ktoolbox.toml`. Для общих правил используйте `scope.mode = "global"`, для выбранных авторов — `scope.mode = "creators"` с точными `service:id`. Правила проверяются в порядке файла и останавливаются на первом совпадении.

Перед длительным запуском проверьте регулярные выражения и области:

```bash
ktoolbox config validate
```

## Почему в перенаправленном журнале прогресс выглядит иначе?

Интерактивный прогресс Rich используется только в терминале. Он показывает скорость и ETA каждого активного файла и общую скорость всех загрузок в строке `Files`. Каналы, CI, `NO_COLOR` и `--plain` используют стабильный построчный вывод, чтобы сообщения не повреждали область ANSI. `--no-color` сохраняет интерактивную компоновку без цвета, а `--quiet` скрывает прогресс и обычные журналы.

## Почему WebUI не запускается?

Установите `ktoolbox[webui]` и передайте каталог проекта. Настройка аккаунта необязательна: без неё терминал выводит имя `admin` и случайный пароль для этого запуска. Явно заданный хеш всё равно должен быть корректным Argon2. Отсутствующий `ktoolbox.toml` создаётся после предупреждения, а блокировка проекта по-прежнему отклоняет второй процесс WebUI для того же проекта.

## Безопасно ли открывать WebUI в сети?

Стандартный сервер использует обычный HTTP, поэтому `0.0.0.0` подходит только для доверенной LAN. Для локальной работы привяжите `127.0.0.1`, для удалённой — разместите службу за обратным прокси HTTPS. Аутентификация, CSRF, строгие cookie, ограничение запросов и защитные заголовки защищают приложение, но не шифруют путь HTTP.

## Что происходит с задачей WebUI после перезапуска?

Задачи в очереди остаются там. Выполнявшиеся задачи отмечаются `interrupted`, а не запускаются молча заново, поскольку их конфигурация и удалённое состояние могли измениться. Проверьте и продолжите их вручную. Завершённые файлы и возобновляемые временные файлы сохраняются.

## Может ли удаление задачи WebUI удалить посторонние файлы?

Обычное удаление стирает только историю задачи. Удаление вывода требует предварительного просмотра и подтверждения, затем проверяет записи владения и метаданные файлов. Существующие, изменённые, общие, не обычные файлы и символические ссылки пропускаются.

## Как узнать настоящую причину сбоя синхронизации?

Откройте неудачную задачу и раздел **Почему задача завершилась ошибкой**. KToolBox указывает автора или файл, этап сбоя, возможность повтора и рекомендуемое действие. Несовместимый формат ответа обычно означает изменение формы поля Pawchive: обновите KToolBox и сообщите безопасное имя операции и путь поля из панели. Сетевые ошибки, тайм-ауты и ограничения частоты могут исчезнуть при повторе. Отчёт очищен и никогда не содержит тело ответа вышестоящего сервера, название работы, Cookie или полный URL загрузки.

## Обязателен ли uvloop или winloop?

Нет. Это необязательные оптимизации цикла событий. Используйте `ktoolbox[uvloop]` в Linux/macOS или `ktoolbox[winloop]` в Windows. Без них KToolBox продолжит работу со стандартным циклом asyncio Python.

## Почему антивирус может отметить упакованный исполняемый файл?

Некоторые эвристические сканеры отмечают пакеты PyInstaller или менеджеры загрузок. Выпуски собираются публичной автоматизацией репозитория; также можно установить пакет через `pipx` или проверить и собрать исходный код локально.
