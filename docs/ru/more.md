# Сведения о проекте

## Ветка разработки

До готовности стать основной линией выпуска Pawchive v1 разрабатывается в ветке [`pawchive`](https://github.com/Ljzd-PRO/KToolBox/tree/pawchive).

Изменения разделены на целевые коммиты, посвящённые контракту, клиенту, миграции проекта, тестам, документации и метаданным выпуска. Исходный файл OpenAPI Pawchive остаётся неизменным, чтобы изменения созданного кода можно было проверить относительно нормализованного контракта.

## Политика качества

Стандартный набор тестов работает полностью автономно и блокирует случайный доступ к сети. Написанный вручную слой API должен сохранять 100 % покрытия строк и ветвей, созданные модели исключаются из статистики, а покрытие всего проекта не должно опускаться ниже 85 %.

CI также проверяет документ OpenAPI, детерминированное создание моделей, Ruff, строгий Mypy для слоя API, компиляцию байт-кода Python, сборку пакетов и строгую сборку MkDocs на поддерживаемых версиях Python.
