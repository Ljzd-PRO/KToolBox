# Руководство по командам

KToolBox использует команды Cyclopts с обычными параметрами через дефис и справкой Rich. Справка выводится напрямую и никогда не открывает программу просмотра.

```bash
ktoolbox --help
ktoolbox sync --help
```

![Обзор команд KToolBox](../../assets/cli-overview.png)

Глобальные параметры могут находиться до или после команды:

```bash
ktoolbox --config ./ktoolbox.toml --plain sync
ktoolbox creator list --json --config ./ktoolbox.toml
```

Используйте `--verbose` для диагностических журналов, `--quiet` для скрытия хода выполнения и обычных журналов, `--plain` для стабильного построчного отображения прогресса, а `--no-color` — для сохранения интерактивной компоновки без цветов ANSI.

## Скачивание одной публикации

Передайте URL страницы Pawchive либо три значения идентификации:

```bash
ktoolbox download https://pawchive.pw/fanbox/user/6570768/post/1836570

ktoolbox download \
  --service fanbox \
  --creator-id 6570768 \
  --post-id 1836570 \
  --output downloads
```

С помощью `--revision-id` можно выбрать редакцию. Поскольку у Pawchive нет адреса сведений об одной редакции, KToolBox получает список редакций и находит указанный ID. Задайте `KTOOLBOX_JOB__INCLUDE_REVISIONS=True`, чтобы при скачивании текущей публикации включить все редакции.

После прерывания снова выполните ту же команду. Завершённые файлы пропускаются, а совместимые временные файлы продолжаются с помощью запросов HTTP Range. Явная команда `download` не применяет правила исключения синхронизации.

## Синхронизация авторов

`sync` принимает любое количество URL авторов, идентификаторов `service:id` или псевдонимов из списка проекта:

```bash
# Ограниченная синхронизация одного автора.
ktoolbox sync fanbox:123 --length 1

# Несколько авторов используют общий параллельный пул загрузок.
ktoolbox sync fanbox:123 patreon:456 studio-c --length 10
```

Без `--length` обрабатываются все доступные страницы. На границе CLI `--offset` означает индекс публикации; KToolBox преобразует его в смещения страниц Pawchive по 50 элементов.

```bash
ktoolbox sync fanbox:123 --offset 10 --length 5
ktoolbox sync fanbox:123 --start-time 2025-01-01 --end-time 2025-03-01
```

Каждый автор хранится как `Creator name [service-creator_id]`. Одновременно работают не более `job.creator_concurrency` производителей авторов. Их ограниченные очереди поступают в единый справедливый циклический планировщик, а `job.count` управляет параллельностью передачи файлов. Загрузки начинаются сразу после появления первых заданий, не дожидаясь всех авторов.

Ошибка одного автора не удаляет завершённую работу других. Для неудачных авторов сохраняется прежний индекс, а итоговая команда после вывода сводки завершается с кодом `1`.

## Управление списком авторов

```bash
ktoolbox creator add fanbox:123 --alias studio-a
ktoolbox creator add https://pawchive.pw/patreon/user/456 --alias studio-b
ktoolbox creator disable studio-b
ktoolbox creator list
ktoolbox creator enable studio-b
ktoolbox creator remove studio-b
```

Выполните `ktoolbox sync` без целей, чтобы синхронизировать все включённые записи списка. Явно указанный отключённый автор всё равно выполняется. Идентификаторы `service:id` уникальны, псевдонимы необязательны и уникальны, а при записи конфигурации сохраняются комментарии.

## Исключение публикаций, не являющихся работами

Определите упорядоченные правила полей в `ktoolbox.toml`. Они могут применяться глобально или только к выбранным авторам `service:id` и проверять заголовки, содержимое, теги, имена файлов, ID и вложенные пути списков. Первое совпавшее правило исключает публикацию до создания сведений, редакций, метаданных, каталога или задания на скачивание.

Полные примеры приведены в [руководстве по конфигурации](../configuration/guide.md#post-blockers). `KTOOLBOX_JOB__KEYWORDS_EXCLUDE` сохранён только для миграции как устаревшее глобальное правило заголовка.

## Просмотр публичных данных

```bash
ktoolbox creator search --name example --service fanbox
ktoolbox post search --creator-id 6570768 --service fanbox --query preview --offset 0
ktoolbox post show fanbox 6570768 1836570
```

По умолчанию команды запросов используют компактные таблицы Rich. Добавьте `--json` для машиночитаемого stdout; журналы и ход выполнения остаются в stderr. `--dump path.json` дополнительно записывает проверенные модели в файл.

## Инструменты конфигурации

```bash
ktoolbox config path
ktoolbox config validate
ktoolbox config example
ktoolbox config edit
ktoolbox site-version
```

Для редактора нужна необязательная зависимость `urwid`. Он изменяет как параметры dotenv, так и проектный документ со списком авторов и правилами, затем проверяет их перед сохранением.

## Запуск необязательной панели проекта

После установки `ktoolbox[webui]` привяжите панель к каталогу проекта. Можно настроить аккаунт заранее либо использовать имя `admin` и случайный пароль, напечатанные в терминале для этого запуска. Отсутствующий `ktoolbox.toml` создаётся автоматически после предупреждения:

```bash
ktoolbox webui . --host 127.0.0.1
```

Панель предоставляет те же процессы скачивания, синхронизации, управления списком, правилами, запросами и конфигурацией как управляемые действия проекта, с постоянным прогрессом задач и элементами управления. Перед прослушиванием сетевого интерфейса ознакомьтесь с [руководством по WebUI](../webui.md).

## Код завершения

| Код | Значение |
| --- | --- |
| `0` | Команда успешно завершена. |
| `1` | Ошибка удалённой операции, автора или файла; частичные файлы сохранены. |
| `2` | Недопустимые аргументы или конфигурация. |
| `130` | Пользователь прервал команду. |

Ожидаемые ошибки выводятся без трассировки Python.
