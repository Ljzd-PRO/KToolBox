# Переход на v1

KToolBox v1 содержит несовместимые изменения сервера и библиотечного API.

## Перед обновлением

1. Создайте резервную копию файла `.env` или `prod.env`.
2. Завершите или отмените все выполняющиеся загрузки v0.
3. Перед полной синхронизацией проверьте одного автора с ограничением `--length=1`.

## Обязательные изменения

| Поведение v0 | Поведение v1 |
| --- | --- |
| Серверы Kemono/Coomer | Только Pawchive |
| Python 3.8+ | Python 3.10-3.14 |
| `KTOOLBOX_API__SESSION_KEY` | `KTOOLBOX_DOWNLOADER__SESSION_KEY` |
| Серверы API и файлов смешаны в настройках API | Серверы API и статики в `api`; файловый сервер в `downloader` |
| Запрос сведений о редакции | Получить список редакций, затем выбрать по `revision_id` |
| Ответы-обёртки, например `.data.post` | Непосредственно типизированные `Post`, `Revision` и другие модели Pydantic |
| Интерфейс команд Python Fire | Дерево команд Cyclopts с параметрами через дефис и явными кодами завершения |
| Один автор на синхронизацию | Любое количество целей или включённый список проекта |
| Только глобальный `keywords_exclude` | Упорядоченные глобальные и привязанные к автору правила `field-match` |

Новые значения по умолчанию:

```dotenv
KTOOLBOX_API__NETLOC=pawchive.pw
KTOOLBOX_API__STATICS_NETLOC=pawchive.pw
KTOOLBOX_API__PATH=/api/v1
KTOOLBOX_DOWNLOADER__FILES_NETLOC=file.pawchive.pw
KTOOLBOX_DOWNLOADER__FILE_PATH_PREFIX=/data
```

## Команды CLI

Скрытые псевдонимы временно сохраняют работоспособность автоматизации, но при каждом вызове выводят предупреждение об устаревании:

| Команда v0 | Команда v1 |
| --- | --- |
| `download-post` | `download` |
| `sync-creator` | `sync` |
| `search-creator` | `creator search` |
| `search-creator-post` | `post search` |
| `get-post` | `post show` |
| `config-editor` | `config edit` |
| `example-env` | `config example` |

Параметры отображаются как `--creator-id`, а не с подчёркиваниями в стиле Python. Старое написание с подчёркиваниями по-прежнему принимают псевдонимы совместимости. Справка выводится напрямую и больше не требует выхода из программы просмотра.

Ошибки CLI теперь используют код процесса: `0` — успех, `1` — ошибка удалённой службы, автора или загрузки, `2` — ошибка аргумента или конфигурации, `130` — прерывание. JSON и таблицы выводятся в stdout, ход выполнения и журналы — в stderr.

## Список авторов и правила исключения

Создавайте `ktoolbox.toml` только для многократно используемого списка авторов или структурированных правил исключения. Отсутствующий файл означает допустимый пустой проект.

```toml
schema_version = 1

[[creators]]
service = "fanbox"
creator_id = "123"
alias = "studio-a"
enabled = true
```

Перенесите непустые значения `KTOOLBOX_JOB__KEYWORDS_EXCLUDE` в глобальное условие заголовка `field-match`. Старая настройка остаётся активной как неявное правило и выводит предупреждение, но KToolBox не переписывает локальные файлы. См. [руководство по конфигурации](configuration/guide.md#post-blockers).

`KTOOLBOX_JOB__CREATOR_CONCURRENCY` по умолчанию равен `4` и ограничивает производителей для авторов. Существующий `KTOOLBOX_JOB__COUNT` продолжает ограничивать обработчики файлов.

## Необязательная WebUI

В v1 добавлена новая панель HeroUI; она не переносит и не использует прежнюю экспериментальную ветку `webui`. Установите `ktoolbox[webui]`, выберите каталог проекта и настройте новую единственную учётную запись. Отсутствующий `ktoolbox.toml` создаётся автоматически после предупреждения; стандартные учётные данные не создаются.

```bash
ktoolbox webui hash-password
ktoolbox webui /path/to/project --host 127.0.0.1
```

`.env` и `prod.env` теперь являются игнорируемыми локальными файлами, а не примерами под управлением версий. Храните в них учётные данные и сеансы загрузчика, используйте `example.env` как общедоступный шаблон и перед обновлением проверьте старые отслеживаемые dotenv-файлы. WebUI создаёт `.ktoolbox/webui.sqlite3` и блокировку проекта; ни то ни другое не меняет форматы вывода загрузок CLI.

Риски развёртывания HTTP и поведение постоянных задач описаны в [руководстве по WebUI](webui.md).

## Библиотечный API

Прежние `BaseAPI`, средства вызова классов, функции `get_*` уровня модуля, `APIRet` и обёртки ответов Kemono удалены без псевдонимов совместимости. Используйте экземпляр асинхронного клиента:

```python
import asyncio

from ktoolbox.api import PawchiveClient


async def main() -> None:
    async with PawchiveClient() as client:
        profile = await client.get_creator_profile("fanbox", "6570768")
        posts = await client.list_creator_posts(profile.service, profile.id, offset=0)
        print(profile.name, len(posts))


asyncio.run(main())
```

Успешные вызовы возвращают модели Pydantic v2. Ошибки вызывают типизированные подклассы `PawchiveError`; см. [документацию API](api.md).
