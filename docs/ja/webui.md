# WebUI

KToolBox WebUI は React と HeroUI で構築された、プロジェクトに結び付く管理パネルです。CLI と同じ設定を編集し、同じ Python サービスを呼び出します。CLI サブプロセスの起動や解析はしません。タスク、試行、ログ、所有記録は選択したプロジェクト内の `.ktoolbox/webui.sqlite3` に永続化されます。

## インストールと起動

オプションのランタイムをインストールし、プロジェクトディレクトリを作成します。

```bash
pipx install "ktoolbox[webui]" --force
mkdir ktoolbox-project
cd ktoolbox-project
```

起動時の認証情報は省略できます。未設定の場合、そのプロセスで使う `admin` と新しいランダムパスワードがターミナルに表示されます。固定の認証情報には、非表示入力で Argon2id ハッシュを生成します。

```bash
ktoolbox webui hash-password
```

アカウントをプロジェクトの `.env` に保存します。シェル形式の `$` をそのまま残すため、ハッシュを引用符で囲みます。

```dotenv
KTOOLBOX_WEBUI__USERNAME=owner
KTOOLBOX_WEBUI__PASSWORD_HASH='$argon2id$v=19$...'
```

プロジェクト用パネルを起動します。

```bash
ktoolbox webui .
ktoolbox webui . --host 127.0.0.1 --port 8789 --no-open
```

既定値は `0.0.0.0:8789` で、ローカルブラウザが自動で開きます。`--host`、`--port`、`--no-open` はそのプロセスの環境設定を上書きします。`ktoolbox.toml` がない場合は、警告を表示して最小の有効な文書をアトミックに作成します。認証情報がなくても起動でき、空のユーザー名は `admin`、両方のパスワードが空なら今回用の新しいパスワードを生成してターミナルに表示します。

## セキュリティモデル

KToolBox のローカル WebUI アカウントは 1 つです。明示設定が優先され、`KTOOLBOX_WEBUI__PASSWORD_HASH` は平文の `KTOOLBOX_WEBUI__PASSWORD` より優先されます。どちらも未設定なら起動ごとにメモリ上でパスワードを生成し、有効なユーザー名とともにそのターミナルだけへ表示します。安定運用ではハッシュを設定し、両 dotenv ファイルをバージョン管理から除外してください。

セッションはランダムな不透明トークンを使います。SQLite はトークンハッシュだけを保存します。ブラウザ Cookie は `HttpOnly` と `SameSite=Strict` で、HTTPS では `Secure` になります。変更リクエストにはセッション別 CSRF トークンと同一オリジン確認が必要です。ログイン試行はレート制限され、API レスポンスはキャッシュされず、厳格なコンテンツ、フレーム、リファラー、ブラウザ権限ヘッダーを送信します。

内蔵サーバーは HTTP を使います。既定の LAN リスナーは信頼できるネットワーク専用で、そうでなければパスワード、Cookie、パス、ログ、設定が通信中に見えます。1 台では `--host 127.0.0.1`、リモートでは信頼できるリバースプロキシで HTTPS を終端し、ネットワークアクセスを制限してください。安全でないページではログインページとアプリシェルに HTTP 警告が残ります。

同時に 1 つのスケジューラーだけがプロジェクトを開けます。プロジェクトロックが 2 つの WebUI プロセスによるキューと出力の競合を防ぎます。

## プロジェクトのワークフロー

初回はブラウザ言語に従い、簡体字中国語、繁体字中国語、英語、日本語、韓国語、フランス語、ロシア語の選択を保持します。言語を切り替えると、React Aria の日付、数値形式、自然順ソート、設定メタデータ、入力検証、既知のサーバーエラーも同時に更新されます。ライトまたはダークを選ぶまでは OS のテーマに従います。青、エメラルド、紫、ローズ、アンバーのアクセントを選べますが、有効なフォームスイッチは全パレットで青のままです。デスクトップはコンパクトなサイドバー、狭い画面は Drawer を使います。

![7 言語セレクター](../assets/webui/23-language-menu-seven-locales.png)

![ライト設定エディター](../assets/webui/09-configuration-light.png)

編集領域は控えめな第 2 サーフェスと明確なフィールド背景を使います。アイコンで確認しやすくし、スイッチとチェックボックスは中央の操作ボタンのようにせずラベルと左揃えにします。スイッチはオフがグレー、オンが青で、チェックボックスは選択または不確定時だけ印を表示します。編集モーダルと固定アクションバーは連続したサーフェスを共有します。

主な領域：

- **概要：**プロジェクトパス、キュー状態、アクティブ転送合計、最近のタスク。
- **タスク：**同期または単一作品ダウンロードの作成、編集、一時停止、再開、停止、再実行、削除、確認。行をクリックすると詳細を開き、複数選択で対応する一括操作を実行できます。
- **クリエイター：**Pawchive の検索と一覧項目の追加、メモ編集、有効化、無効化、削除。一括での有効化、無効化、削除にも対応します。
- **投稿：**リモートメディアや本文を展開せずに検索、改訂の確認、ダウンロードタスクの作成。
- **除外ルール：**`field-match` の順序とスコープ、ネストした `any`/`all`、包含、等価、正規表現、存在条件の構成。
- **設定：**型付きフォームまたは高度なテキスト表示で `.env`、`prod.env`、`ktoolbox.toml` を編集。
- **システム：**プロジェクトとアプリのバージョン確認、環境例のダウンロード。

![狭い画面のタスクエディター](../assets/webui/19-task-form-mobile-light-zh.png)

![狭い画面のクリエイター一覧](../assets/webui/12-creators-mobile-light-zh.png)

タスク作成はオーバーフロー操作のない 2 つの固定タブを使います。同期日付は公式 HeroUI の 1 つの `year/month/day - year/month/day` 範囲フィールドのままで、「開始日なし」と「終了日なし」はそれぞれの境界を個別に消去します。投稿オフセットは 50 ずつ変更されます。タイトルフィルターは、カンマまたは Enter で作成する削除可能な HeroUI Chip を使います。単一作品のダウンロードと一覧へのクリエイター追加は独立した HeroUI フィールドを使い、`/platform/user/creator/post/post` のようなコード風の Pawchive パス断片で区切ります。区切りは入力フィールドとして模造しません。

デスクトップ行とモバイル項目はどちらもクリエイター ID を先頭に表示します。任意の一覧メモは別に表示され、ID を置き換えません。既存のクリエイターを編集するとき、プラットフォームとクリエイター ID は一覧項目を識別するため、表示されたまま読み取り専用になります。

## 設定の編集

フォームのラベルと説明は Python 識別子ではなく、明示的にローカライズされたテキストです。英語設定クラスの `:ivar field:` docstring は引き続きフィールド意味の基準であり、完全性を検査した言語カタログが 7 言語すべてのラベルと説明を提供します。型、既定値、範囲、シークレット情報は Pydantic が提供します。

`.env` と `prod.env` タブは最終有効値と取得元 Chip を表示します。プロセス環境で上書きされた値は読み取り専用です。シークレットは既定でマスクされます。高度なテキスト編集はシークレットを公開し得るため追加警告を表示します。

保存前にサーバーが提案ファイルを解析・検証し、意味的差分を返します。ETag で古い編集を拒否し、ファイルをアトミックに置換します。TOML エディターは既存の TomlKit/Pydantic ストアを使うため、構造化された一覧とルールを変更してもコメントが残ります。

## タスクのライフサイクル

`sync` と `download` タスクは対応する CLI 入力をすべて保持します。対象なしの同期はタスク作成時に現在の有効な一覧を解決します。各試行は不変で秘匿化済みの設定スナップショットを受け取り、その後の設定変更は将来の試行だけに影響します。

各タスクは正規化対象キー、任意の投稿タイトルとクリエイター名を含む表示専用スナップショットも保存します。オフラインでも読め、実行、重複排除、リソースロックには影響しません。キュー行は出力パスではなく対象を先に表示し、詳細、一時停止/再開、停止、編集、順序、削除の操作を直接表示します。

![読みやすい対象を持つデスクトップタスクキュー](../assets/webui/21-task-queue-1440-dark-zh.png)

![操作を直接表示するモバイルタスクキュー](../assets/webui/22-task-queue-mobile-light-zh.png)

![ダークタスクエディター](../assets/webui/18-task-form-1024-dark-zh.png)

最上位キューは既定で 2 タスク（`KTOOLBOX_WEBUI__MAX_ACTIVE_TASKS`）を実行し、各タスクは設定されたクリエイターとファイルの並行性を維持します。同一のアクティブタスクは既存タスクに解決されます。正規化した出力、クリエイター、投稿が重なるタスクはリソースロックが解放されるまで `blocked` で待ちます。

ライブイベントは再接続対応 SSE を使います。REST タスク状態が正です。詳細画面は準備済みクリエイター、ファイル、バイト、全体進捗、合計とファイル別速度、ETA、スキップ/失敗数、アクティブ項目、構造化ログを報告します。

失敗した試行では、件数だけでなく、上限付きで秘匿化された診断レポートを永続化します。タスク行には最初の有用な原因を表示し、詳細ではクリエイター別・ファイル別に、失敗段階、再試行の可否、安全なフィールドパス、推奨対応を示します。上流レスポンス本文、作品タイトル、Cookie、完全なダウンロード URL は保存しません。狭い画面では 64px のワークバー、12px のページ余白、コンパクトな外観 Popover により、フォーム文字を 16px 未満へ縮小せず情報量を高めます。MCP ツール一覧は折りたたみ可能な HeroUI グループを使い、検索または権限フィルター時に一致グループを自動展開します。

![構造化されたタスク失敗説明](../assets/webui/26-task-failure-1440-light-zh.png)

![コンパクトなモバイル外観操作](../assets/webui/27-appearance-mobile-dark-zh.png)

![ライブタスク進捗](../assets/webui/14-task-running-1024-dark-zh.png)

一時停止は協調的です。アクティブなネットワークストリームを閉じ、完成済みファイルと再開可能な一時ファイルを残し、再開時に新しい試行を作ります。停止は定義を残すため編集・再実行できます。プロセス再起動は以前の実行作業を `interrupted` にし、復旧は必ず明示的です。

通常のタスク削除はキュー記録、試行、ログだけを消します。「出力を削除」はまずファイル数とバイト数をプレビューします。確認後、そのタスクが作成したと記録された未変更の通常ファイルだけを削除し、シンボリックリンク、既存、変更済み、共有ファイルはたどらず削除しません。

## 自動更新

ログイン後は 1 本の SSE 接続だけを使用し、複数のブラウザータブ間でタスク、クリエイター、除外ルール、設定、MCP トークン、現在開いているリモートディレクトリを同期します。構造的な変更は通常 1 秒以内に表示され、タスク進捗はタスク一覧全体を繰り返し取得せず、ローカルのクエリキャッシュを直接更新します。

ライブ接続が 5 秒以上利用できない場合、WebUI はコンパクトな警告を表示し、ローカルプロジェクトデータだけを 10 秒ごとに更新します。SSE が復旧するとフォールバック更新を直ちに停止し、一度だけ全体を更新します。Pawchive 検索、作品詳細、バージョン確認は常にオンデマンドであり、フォールバック処理から要求されません。

システムページには現在の更新方式と最終シグナル時刻が表示され、手動更新と再接続を実行できます。未保存のフォームを編集中に別タブや MCP クライアントがデータを変更した場合、KToolBox は下書きを保持し、再読み込みか編集継続かを選択させます。保存時の ETag とタスク状態の競合チェックも引き続き適用されます。

## WebUI 環境リファレンス

| 変数 | 既定値 | 意味 |
| --- | --- | --- |
| `KTOOLBOX_WEBUI__HOST` | `0.0.0.0` | 待ち受けインターフェース。 |
| `KTOOLBOX_WEBUI__PORT` | `8789` | 1～65535 の待ち受けポート。 |
| `KTOOLBOX_WEBUI__OPEN_BROWSER` | `True` | 起動後にローカル URL を開く。 |
| `KTOOLBOX_WEBUI__USERNAME` | 空 → 起動時 `admin` | 任意の単一アカウント名。 |
| `KTOOLBOX_WEBUI__PASSWORD_HASH` | 空 | 推奨する固定 Argon2id ハッシュ。 |
| `KTOOLBOX_WEBUI__PASSWORD` | 空 → 起動ごとにランダム | 平文の代替。ハッシュがあれば無視。 |
| `KTOOLBOX_WEBUI__MAX_ACTIVE_TASKS` | `2` | 同時最上位タスク、1～16。 |
| `KTOOLBOX_WEBUI__SESSION_IDLE_HOURS` | `24` | 最終利用からのセッション期間。 |
| `KTOOLBOX_WEBUI__SESSION_ABSOLUTE_HOURS` | `168` | ログインからの最大セッション期間。 |

タスク履歴が重要な場合は `ktoolbox.toml`、ローカル dotenv、`.ktoolbox/webui.sqlite3` を一緒にバックアップします。WebUI 実行中にデータベースをコピーしないでください。
